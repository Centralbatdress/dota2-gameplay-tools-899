A lightweight, high-performance telemetry platform and UI overlay toolkit for **Dota 2**. Designed for real-time match state analysis, advanced data visualization, and game mechanics prototyping. 🔥

---

### 📋 Overview

This project provides a modular framework for interacting with the Dota 2 match state, rendering custom overlay interfaces, and reading client telemetry data. Perfect for analytical tracking, streamlining gameplay mechanics testing, and visualizing hidden strategic data directly on your screen. 👁️‍🗨️

---

### ✨ Features

#### 👁️ Advanced Data Visualization
* **Position Tracking & Prediction:** Advanced algorithms to predict and visualize hero positions and map movements.
* **Hidden Particle Analytics:** Real-time logging and rendering of visual effects and indicator alerts (e.g., *Sunstrike*, *Torrent*, and *Ice Blast* logic).
* **Cooldown Telemetry:** Monitor accurate state timers and status logs for abilities and global action availability.

#### 🎨 Custom UI Overlay Framework
* **Interactive HUD Layout:** Sleek, fully interactive and customizable in-game menu framework (Draw/Click GUI).
* **Performance Optimized:** Highly efficient UI processing layer that maintains consistent framerates (FPS-friendly).

#### ⏱️ Match Event Tracking
* **Vision & Ward Analytics:** Immediate diagnostic alerts for vision changes and ward coverage modifications.
* **Objective Timers:** Precise visual indicators for internal neutral camp timings, stacking windows, and Roshan status.

#### 🤖 Automation & Macro Prototyping
* **Input Sequencing:** Configurable utility for multi-key execution sequences on complex heroes (e.g., Invoker, Tinker, Skywrath Mage).
* **Mechanics Optimizer:** Automated micro-assistance for precise last-hitting and structural denial tracking.
* **Instant Reaction Utility:** Automated response triggers for low-latency active item usage based on real-time combat status.

#### ⚡ Lightweight & Secure Architecture
* **Minimal Footprint:** Performance-oriented code structure designed for zero-latency background execution.
* **Stability First:** Engineered with a strong emphasis on low system resource consumption and background client stability.
